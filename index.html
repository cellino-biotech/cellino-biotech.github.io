Cellino software development

A guide to configuring your local environment.

Intro

Prerequisites

The main prerequisites for our development environment are git and pyenv. These tools help manage code repositories and local Python environments, respectively. Each developer also needs to configure gcloud as a way to authenticate access to our cloud services. Package managers, such as homebrew or chocolatey, are highly recommended to streamline installation procedures.

Windows

Chocolatey

If not already installed, it is highly recommended to use chocolatey for general package management in Windows. Open an administrative shell (PowerShell) and download and run the installation script:

Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

Git

Download the git installer and follow the guided installation. Most users prefer the default settings, though you may wish to customize the default text editor and shell application. Many Cellino developers prefer Git Bash, especially if the individual is more familiar with bash and Unix tools.

Pyenv

Though lacking some of the useful features found in pyenv, pyenv-win is a great tool for managing different Python installations on your local machine. Install using chocolatey :

choco install pyenv-win

Now you see why we pushed chocolatey

To install Python distributions, with version 3.9.6 as an example:

pyenv install 3.9.6

Check to see all Python versions on local machine and configure scope settings:

print list of installed Python versions

pyenv versions

set Python 3.7 as the global executable

pyenv global 3.7.8

set Python 3.9 as the local executable for the working directory

pyenv local 3.9.6

Unfortunately, pyenv-win does not support virtual environment management. To create a virtual environment in Windows, first navigate to the venv directory of the project repo and, after switching to the desired Python version, run:

python -m venv <venv-name>

Each time you start a shell, the virtual environment will have to be activated:

projectdir\venv\venv-name\Scripts\activate

Mac

Homebrew

If not already installed, it is highly recommended to use homebrew for general package management in macOS:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Git

With homebrew, install git:

brew install git

Now you see why we pushed homebrew

Linux

Package managers

Each Linux flavor has its own default package manager, e.g., apt, yum, and zypper. This tutorial focuses on Ubuntu-based distributions, which include Mint and Pop_OS!. Reference materials will be linked for installation on other distros.

Git

To install git:

sudo apt-get install git

Pyenv

Similar to macOS, pyenv installation on Linux requires build dependencies and modification of the environment profile. For a manual installation, follow the instructions from RealPython. Alternatively, execute the following command to run our in-house install script. Note that this script only works for users working with bash:

curl https://raw.githubusercontent.com/cellino-biotech/cellino-biotech.github.io/main/scripts/pyenv-install-linux.sh?token=AEJ6ATO54DHOSSWJLONQF5TBNL7OE | bash

With successful completion, the terminal may output a message with instructions to update the PATH variable. This is a default message from the pyenv developers and may be disregarded as these steps were automated in the script. After installation, reset the shell:

exec "$SHELL"

Confirm that pyenv has been installed correctly:

pyenv --version

Install the desired Python version (3.9.6 recommended):

pyenv install 3.9.6

Set the new Python distribution as the global executable:

pyenv global 3.9.6

Confirm that the commands python and pip point to the correct executables:

which python

which pip
